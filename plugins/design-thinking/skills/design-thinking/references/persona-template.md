# Persona Template

## Structure

```markdown
### PERSONA-{SEQ}: {Name} — {Role/Archetype}

**Demographics**
- Age range: {range}
- Role/Title: {role}
- Technical level: {novice|intermediate|advanced|expert}
- Environment: {work context, tools, team size}

**Goals**
1. {Primary goal — what they're trying to achieve}
2. {Secondary goal}
3. {Tertiary goal}

**Frustrations**
1. {Primary frustration — linked to PP-XXX}
2. {Secondary frustration — linked to PP-XXX}

**Behaviors**
- {How they currently work}
- {Tools/workarounds they use}
- {Frequency of task}

**Quotes** (from empathy interview)
- "{Direct quote or synthesized insight}"

**Needs**
- {What they need to succeed}

**Influence on Design**
- {How this persona should shape the solution}
```

## Guidelines

1. **Minimum 1 persona** per session — primary user must always be represented
2. **Maximum 4 personas** — more signals scope creep or poorly defined user base
3. Every persona must link to at least one pain point (PP-XXX)
4. Include both primary (direct users) and secondary (stakeholders) personas
5. Base personas on real data from the empathy interview, not assumptions
6. Mark any speculative attributes with `[INFERRED]`
